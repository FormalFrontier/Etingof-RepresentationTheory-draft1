## Accomplished

### Issue #2055: garnir_identity_expansion analysis (skipped as infeasible)

Conducted deep mathematical analysis of the `garnir_columnInvCount_decrease` general case (the single remaining sorry in the Garnir straightening infrastructure). Added one helper lemma (`YoungSymmetrizer_mul_garnirElement`) but determined the sorry reduction is infeasible with the current proof architecture.

**Helper added:** `YoungSymmetrizer_mul_garnirElement` — proves `c_λ * G = 0` (Young symmetrizer annihilates Garnir element on the right) from `garnir_row_annihilates` using `c_λ = b_λ * a_λ` and associativity.

**Key findings from analysis:**

1. **Swap approach fails**: `garnir_swap_identity` gives `of(σ) * c_λ = -of(σ * swap(p₁,p₂)) * c_λ`, but `columnInvCount'(σ * swap(p₁,p₂))` does NOT always decrease. Counterexample: n=6, partition [3,3], where the swap fixes one column inversion but creates another (net change = 0).

2. **Column subgroup right-multiplication fails**: `of(σ) * c_λ = sign(q) * of(σ*q) * c_λ` for q ∈ Q_λ, but Q_λ cannot always make σ*q column-standard. Counterexample: n=6, partition [3,2,1], where entries from different canonical columns create conflicting ordering constraints that Q_λ cannot resolve.

3. **Garnir element decomposition fails**: From `c_λ * G = 0`, we get `of(σ) * c_λ = -Σ sign(w) * of(σ) * c_λ * of(w)`. But individual terms `c_λ * of(w)` are NOT in V_λ = ℂ[S_n] * c_λ (verified algebraically for n=3, [2,1]). So we cannot apply the induction hypothesis to individual terms.

4. **Root cause**: The `columnInvCount'` measure and the `c_λ = b_λ * a_λ` convention (opposite of James's `a_λ * b_λ`) make the standard straightening proof difficult. The comment at line 836-843 of PolytabloidBasis.lean already identifies this convention mismatch as a "ROOT CAUSE" for the related `column_standard_in_span'` sorry.

**Recommendations for replanning:**
- (a) Use dominance ordering on tabloids instead of `columnInvCount'`
- (b) Follow James Chapter 7 tabloid-based approach (requires TabloidModule infrastructure)
- (c) Restructure to avoid `garnir_columnInvCount_decrease` entirely — possibly prove spanning via character-dimension argument or by changing the Young symmetrizer convention
