## Accomplished

Proved `one_round_or_simpleRoot` in `CoxeterInfrastructure.lean` sorry-free (issue #2114, PR #2115). This was the last sorry in the file.

The lemma threads an indecomposable representation through reflection functors along an admissible ordering σ, returning either:
- A prefix of σ that reduces the dimension vector to a simple root, or
- The Coxeter-transformed representation on Q (after full round returns to original quiver)

Key technical challenges solved:
1. **Typeclass instance conflicts**: Two `Quiver` instances (`Q_cur` and `iteratedReversedAtVertices Q_cur tail`) in scope caused "synthesized type class instance not definitionally equal" errors. Introduced `SurvivingRepData` wrapper to bundle representation data without exposing conflicting instances in the return type.
2. **Dependent type transport**: `heq ▸ ρ_end` failed for individual fields due to dependent types. Solved by transporting the entire `SurvivingRepData` bundle at once.
3. **Typeclass pollution ordering**: `haveI` for `Q_rev` instances poisoned subsequent typeclass lookups for `Q_cur`. Moved `Q_cur`-dependent computations (`hd_eq`, `hbridge`) before `Q_rev` instance introduction.
4. **Universe pinning**: Pinned `QuiverRepresentation` to `{_, 0, 0, 0}` throughout the Coxeter iteration chain in `CoxeterInfrastructure.lean`, `Corollary6_8_3.lean`, and `Theorem6_5_2.lean`.

Also added helper `walk_admissible_ordering` (private inductive lemma) that does the core induction over the vertex list.
