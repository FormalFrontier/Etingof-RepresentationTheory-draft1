## Accomplished

Fixed the polytabloid definition bug (issue #2129, human-oversight):

1. **Added `RelColumnAntisymmetrizer`**: κ_T = Σ_{π ∈ C_T} sign(π) · of(π) where C_T = σ_T⁻¹ Q_λ σ_T is the T-relative column subgroup. Defined by summing over Q_λ and conjugating by σ_T (avoids circular import with TabloidModule.lean which already has RelColumnSubgroup).

2. **Changed `polytabloid` definition**: From `of(σ_T) * YoungSymmetrizer` (= `of(σ_T) * a_λ * b_λ`) to `RelColumnAntisymmetrizer T * of(σ_T) * RowSymmetrizer` (= `κ_T * of(σ_T) * a_λ`). This follows James' original treatment.

3. **Updated `polytabloid_canonical`**: Now proves the canonical polytabloid equals `b_λ * a_λ` (not `a_λ * b_λ` = YoungSymmetrizer).

4. **Sorry'd 4 proofs** that relied on the old definition structure:
   - `polytabloid_mem_spechtModule` — membership in V_λ = ℂ[S_n] · c_λ is no longer trivial
   - `polytabloid_apply` — coefficient formula changes completely
   - `polytabloid_self_coeff` — diagonal entry e_T(σ_T) = 1 needs new proof
   - `polytabloid_support` — support characterization changes

5. **All downstream files compile**: TabloidModule.lean, Theorem5_17_1.lean, etc.

## Current frontier

The polytabloid definition is now mathematically correct. The 4 new sorries need proofs with the new definition. Key insight: the coefficient formula `(polytabloid T)(σ) = (b_λ * a_λ)(σ_T⁻¹ * σ)` may hold (since `of(σ_T) * κ_T = b_λ * of(σ_T)`), but `κ_T * of(σ_T)` has a different structure.
